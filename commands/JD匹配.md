## /JD匹配 — 用具体 JD 检查你的简历匹配度,给针对性改写建议

用法:
- `/JD匹配 inputs/简历.docx` — 然后粘贴 JD 内容
- `/JD匹配 inputs/简历.docx inputs/JD.txt` — 直接传两个文件
- `/JD匹配 [直接粘贴 JD 链接或内容]` — 用 profile.json 里登记的简历

### 这个命令是干嘛的

很多人投简历不针对 JD 改,导致**用 1 份通用简历投 50 个岗,转化极低**。本命令做的事:

1. 分析 JD 的核心需求(关键技能 / 关键词 / 经历偏好)
2. 评估你简历的匹配度(覆盖了哪些 / 漏了哪些 / 表达不到位的)
3. 给出**针对这份 JD 的具体改写建议**(改哪条 bullet / 加哪个关键词 / 重排哪段)
4. 给一个匹配度评分(0-100)
5. 询问你要不要直接基于建议改出新版 docx

### 流程

1. **读 skill 资源**:
   - `~/.claude/skills/industry-resume-toolkit/SKILL.md`
   - `patterns/通用.md`
   - 用户对应企业性质的 patterns(从 profile.json 读)
   - 用户对应岗位的 patterns(如有)

2. **读 profile.json**(在 `~/.claude/skills/industry-resume-toolkit/profile.json`)

3. **读简历**(`$ARGUMENTS` 或 profile 里登记的简历路径)

4. **拿 JD**:
   - 用户在命令里贴了 JD 文本 → 直接用
   - `$ARGUMENTS` 是文件路径 → 读文件
   - 都没有 → 提示用户:"把 JD 完整内容粘给我"

5. **分析 JD,提取**:
   - **硬性要求**:学历 / 专业 / 工作年限 / 必备证书 / 必备工具
   - **核心职责关键词**(JD 中反复出现的动词 + 名词)
   - **加分项**(JD 中"加分"、"优先"标记的)
   - **企业价值观信号**(eg 字节"始终创业" / 阿里"客户第一")

6. **对照简历评估**,分类标注:
   - **覆盖项**:简历里已经有 X 关键词 / 经历 / 技能
   - **缺失项**:JD 要 X,简历里没体现
   - **错配项**:简历里有内容但跟 JD 不在一个频道
   - **可挖项**:简历里**可能有但表达不到位**(eg 你做过 AB 测试但没写)

7. **输出格式**(在对话里展示):

```markdown
# JD 匹配分析 — 目标:[岗位 @ 公司]

## JD 关键需求(我从 JD 提炼的)
**必备**:
- 学历:本科及以上 [✓ 你符合 / ✗ 你不符合]
- 工作年限:3-5 年
- 工具:Excel(熟练)、SQL(掌握)、Tableau(加分)

**高频关键词**:用户增长(出现 5 次)、数据分析(4 次)、AB 测试(3 次)、跨部门协作(3 次)

**加分项**:
- 大厂背景
- 内容运营经验

## 你简历的匹配度评估

### ✅ 已覆盖(强项)
- 大厂背景(腾讯音乐)— 直接命中加分项
- 数据透视表 / Excel — 命中"工具"

### ⚠️ 表达不到位(可挖,不用编)
- 你写"通过用户画像优化策略" → JD 要"用户分层 / AB 测试"
  改成:"基于用户分层模型(活跃度/频次/停留时长)设计 AB 测试,验证内容分发策略,转化率提升 18%"

### ❌ 缺失(JD 要但你简历真没有)
- SQL — 你简历完全没提
- 跨部门协作 — 没有具体案例

## 匹配度评分

**74/100** — 整体匹配度中等偏上。强项是大厂背景和数据工具,弱项是缺 SQL 和明确的跨部门案例。

## 改写建议(具体动作)

1. **腾讯音乐那段加 SQL**:你真用过 SQL 查数据吗?有就明确写出来(eg "使用 SQL 查询用户行为数据,日均处理 5w+ 条")
2. **抖音那段改成 AB 测试叙事**:你之前的"提升用户参与度 23%"可以重写成"通过 AB 测试验证内容策略,实验组参与度比对照组高 23%"
3. **加一段跨部门协作具体案例**:找你简历里跟其他团队合作的经历,改成 STAR 法

---

要不要我**基于以上建议直接帮你改出一版新简历**?(我会问你 1-2 个具体问题来填补"可挖项",然后输出 .docx)
- 回 **"改"** → 我开始改,跟 `/改简历` 流程类似但更聚焦这份 JD
- 回 **"先不改"** → 你看完建议自己改
```

8. **简短汇报**:1) 匹配度评分 2) 最关键 3 个改动 3) 是否要直接改成新版

### 重要

- **不要让用户编没做过的事** — 严格遵守保真原则
- **可挖项** vs **缺失项**要区分开(前者是表达问题,后者是真没有)
- JD 关键词提取要准 — 反复出现 / 加粗 / 必备项才算
- 用第二人称称呼用户(eg "你简历的优势")
- 如果用户要"直接改成新版",转到 `/改简历` 流程但**聚焦这份 JD** 而不是泛改
