# 设计 Patterns

涵盖:**UI 设计 / UX 设计 / 视觉设计 / 产品设计 / 交互设计 / 平面设计 / 工业设计 / 品牌设计 / 用户研究**。设计师简历跟其他岗位**结构完全不同**——核心是**作品集**,简历是辅助。

---

## 风格关键词

**视觉表达 / 用户洞察 / 项目完整性 / 设计思维 / 跨部门协作**

招聘官看设计师**先看作品集再看简历**。简历的功能是:**简洁地告诉对方你是谁 + 引导他们去看作品集**。

---

## 核心规则

### 规则 1:作品集链接放最显眼位置

设计师简历**头部必须有作品集链接**(不可省略):

- **个人网站**:[yourname.com](url) 或 [behance.net/yourname](url)
- **Behance / Dribbble**(国际设计师)
- **Zcool 站酷**(国内视觉 / 平面设计师)
- **UI 中国 / Notion 个人页面**(产品设计师)
- **小红书 / Instagram 创意账号**

放在名字下面 + 联系方式上面,**让 HR 一眼看到**。

### 规则 2:项目要"用户问题 → 设计方案 → 业务结果"结构

设计师作品集的每个项目都要讲完整故事,简历里也要简化体现:

> **项目**:[项目名] / [客户]
> **用户问题**:[一句话,谁的什么痛点]
> **设计方案**:[你的解决方法 + 设计原理]
> **结果**:[业务指标 + 用户反馈 + 上线情况]

示例:
> "**[字节] 抖音电商商家后台改版**(2024)
> **用户问题**:中小商家上架商品平均耗时 12 分钟,流程复杂导致 30% 中途流失
> **设计方案**:重构信息架构 + 智能填充 + 模板化预设
> **结果**:**平均耗时降至 4 分钟 (-67%),中途流失率从 30% 降至 8%,商家 NPS 提升 28 分**"

### 规则 3:量化必须"用户级"

设计师的量化指标:
- **用户体验**:NPS / SUS Score / 满意度 / 推荐意愿
- **行为指标**:任务完成率 / 完成时长 / 错误率 / 跳出率
- **业务指标**:转化率 / 留存 / 复购 / GMV
- **设计产出**:覆盖页面数 / 复用组件数 / 设计系统模块

设计师常被认为"无法量化",但**好的设计一定能挂到业务指标**。

### 规则 4:工具栈要"设计师专属 + 分层"

设计师简历的工具栈分类:

```
设计工具(精通 2-3 个):
  - UI / 产品设计:Figma(精通)、Sketch、Adobe XD
  - 视觉 / 平面:Photoshop、Illustrator、InDesign
  - 3D / 动效:After Effects、Cinema 4D、Blender、Lottie
  - 原型 / 交互:ProtoPie、Principle、Origami Studio
  - 工业设计:Rhino、KeyShot、SolidWorks

设计系统 / 协作(熟练):
  - 设计系统:Figma Variants、Tokens、Storybook
  - 协作:Figma、Notion、飞书、Jira

用户研究(熟练):
  - 工具:Maze、UserTesting、Useberry
  - 方法:用户访谈、问卷、可用性测试、卡片分类

前端基础(掌握加分):
  - HTML / CSS / JavaScript 基础
  - 能用 Webflow / Framer 做静态页面
```

### 规则 5:Design Process / 方法论

简历要体现**你不是只会画图,而是有完整设计思维**:

- **Double Diamond / Design Thinking**
- **用户旅程地图 / Journey Mapping**
- **用户画像 / Persona**
- **可用性测试 / Usability Testing**
- **A/B 测试 + 数据分析**
- **设计系统建设**

不只写动作(eg "做了 X 页面"),要写过程(eg "通过用户访谈识别核心痛点 → A/B 测试验证方案 → 上线后追踪 NPS")。

### 规则 6:细分方向要清晰

设计岗细分很细,简历要表明**主攻方向**:

- **UI 设计**:界面视觉 / 设计系统
- **UX 设计 / 交互设计**:用户体验 / 信息架构 / 交互流程
- **产品设计**:UI + UX 综合 / 端到端设计
- **视觉 / 平面设计**:Branding / 海报 / Marketing material
- **品牌设计**:品牌策略 + VI 系统 + Brand Guidelines
- **工业设计**:硬件外观 + 结构 + 用户使用场景
- **用户研究 / UX Research**:定性 / 定量研究

一份简历最多**主推 1-2 个方向**,别全都写。

### 规则 7:实习含金量分级

**T0(顶级设计岗)**:
- **大厂设计中心**:Apple HID / Google Material / 阿里 UED / 腾讯 ISUX / 字节 UX / 微软设计中心 / IBM Design
- **顶级设计咨询**:IDEO / Frog / Smart Design / Method / 洛可可

**T1(优秀)**:
- 大厂业务线设计部门
- 知名 SaaS 公司设计团队(Notion / Figma / Webflow 等)
- 大型品牌的内部设计部门(Nike / Adidas / 字节品牌部)

**T2(中等)**:
- 中型互联网公司设计团队
- 中型设计公司 / 工作室

**包装重点**:实习等级低的,**作品集质量 + 项目完整度比 title 重要**。

### 规则 8:学校 + 学位

设计行业学校认知度:
- **国内 T0**:清华美院 / 中央美院 / 中国美院 / 同济设计学院 / 江南大学设计学院 / 北服 / 央美城市设计学院
- **海外 T0**:Royal College of Art (RCA) / Parsons / RISD / Pratt / CCA / SCAD / UAL / Politecnico di Milano
- **学位类型**:M.Des / MFA / MA in Design / BFA / MArch / MBA + Design 双学位

## 不同企业性质下的设计写法差异

| 企业性质 | 看重什么 | 写法重点 |
|---------|---------|---------|
| **大厂(BAT / 字节)** | 数据驱动 + 设计系统 + 跨团队 | UX 数据、设计系统贡献、AB 测试 |
| **创业公司 / 独角兽** | 全能 + 端到端 + 设计 + 落地 | 0-1 项目、独立 owner、设计开发协作 |
| **设计咨询(IDEO 等)** | 用户研究 + 设计思维 + 跨行业 | 服务设计、Design Thinking、客户多样性 |
| **品牌方(消费品)** | 品牌感 + 视觉感知 + Campaign | 品牌设计、视觉系统、Marketing 协作 |
| **B 端 SaaS** | 复杂场景 + 系统化 + 业务理解 | 设计系统、复杂工作流、企业级 UX |

## 常见地雷

1. **作品集链接缺失 / 死链** — 设计师没作品集 = 没简历
2. **作品集质量不行** — 简历再好作品集烂等于白搭
3. **简历像"作品列表"** — 罗列项目但每个都简短一句话 → 招聘官无法判断你的深度
4. **过度技术化** — 设计师堆 30 个工具 + 5 种编程语言 → 没主线
5. **没有量化** — 设计师常觉得不能量化,但能量化才显专业
6. **不写设计过程** — 只看到 deliverable,看不到 thinking
7. **作品集风格混乱** — 这 5 个项目像 5 个人做的 → 没有 personal aesthetic

## 校招 / 实习设计师特别建议

- **作品集是入场券**:大学期间至少做 3-5 个完整 case study
- **学校项目包装成作品集**:学校课程项目按"用户问题 → 方案 → 结果"重写
- **副业 / 自主项目**:自己做 App / 网站 / 海报 → 加进作品集
- **设计比赛获奖**:K-Design Award / iF / Red Dot / 站酷 / UI 中国比赛
- **设计社团 / 学生项目**:参与学生品牌设计 / 海报设计 → 加进作品集
- **跟开发协作**:能写出"跟开发对齐 / push 设计稿落地"的经验加分
