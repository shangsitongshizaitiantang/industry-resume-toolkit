# 运营 Patterns

涵盖:**内容运营、用户运营、活动运营、社区运营、电商运营、短视频运营、私域运营**等所有运营子方向。运营岗是互联网 / 新消费 / 内容平台的核心岗位之一,跟产品、市场紧密配合但各有侧重。

---

## 风格关键词

**用户视角 / 数据驱动 / 内容感知 / 业务结果 / 迭代速度**

招聘官看运营简历最在意的是:**你懂不懂用户、有没有数据 sense、能不能快速迭代**。

---

## 核心规则

### 规则 1:小标题必须命中"岗位关键词"

招聘官 1 秒看完整段经历靠的是加粗小标题。每段实习/项目至少一个加粗小标题,用 **4-6 字命中 JD 高频词**:

- "高效激活用户参与度" → ❌ 太长
- "**内容策划**" / "**用户分层**" / "**活动复盘**" / "**社群运营**" → ✅ 精准

**不同子方向的高频小标题**:
- 内容运营:内容策划 / 内容分发 / 选题规划 / 爆款拆解
- 用户运营:用户分层 / 留存提升 / 召回唤醒 / 用户画像
- 活动运营:活动策划 / 活动复盘 / ROI 测算 / 节点营销
- 社区/私域:社群运营 / 用户共创 / 私域沉淀 / 转化漏斗
- 电商运营:店铺管理 / 选品测款 / GMV 增长 / 大促复盘
- 短视频运营:账号孵化 / 内容矩阵 / 涨粉策略 / 流量分发

### 规则 2:方法论后面加括号说明

运营岗最有信号的写法是"方法论 + 具体内容"组合:

- ❌ "采用用户分层模型"
- ✅ "用户分层模型(基于活跃度、消费频次、停留时长)"

- ❌ "运用 AARRR 模型"
- ✅ "AARRR 模型(获取 → 激活 → 留存 → 推荐 → 收入)落地用户增长方案"

- ❌ "深度分析用户喜好"
- ✅ "深度分析用户喜好(rap 爱好者、被点名参与者、活跃打赏用户)"

括号里写出具体维度,显得**专业且真做过**。

### 规则 3:动作 + 落地 + 量化三件套

运营每条经历都用这个结构:

> **[动作]**:你做了什么(具体动词 + 名词)
> **[落地]**:怎么落地的(用什么方法 / 工具 / 流程)
> **[量化]**:量化结果(数字或者明显趋势)

示例:
> "策划 12 场用户成长直播主题活动,通过用户分层模型(活跃度+频次)定向触达 5w+ DAU,平均观看时长提升 38%、活动期间留存率提升 23%。"

### 规则 4:量化必须具体

运营岗简历的量化关键指标:
- **量级**:用户数 / DAU / MAU / GMV / 订单数 / 内容曝光 / 阅读量 / 粉丝数
- **效率**:提升幅度 / 时间缩短 / 成本下降 / ROI
- **覆盖**:覆盖多少品类 / 多少团队 / 多少地区

**禁用词**:"很多"、"诸多"、"丰富"、"大量"、"多个"(除非接具体数字 eg "20+")。

### 规则 5:项目逻辑包装(前期/中期/末期)

社团活动 / 校园经历用这个结构包装成"运营项目":

> **前期(策划)**:用户调研 + 内容选题 + 资源协调
> **中期(执行)**:内容产出 + 活动执行 + 数据监测
> **末期(复盘)**:数据复盘 + SOP 沉淀 + 经验传承

让校园经历看起来像真的运营项目。

### 规则 6:工具栈要"运营人专属"

运营岗简历的工具栈应该体现"懂数据 + 会做内容 + 能协作":

- **数据**:Excel(数据透视表)、SQL(基础查询)、Tableau / Power BI / 神策 / GrowingIO
- **内容**:剪映 / Photoshop / Figma(海报)、秀米 / 135 编辑器(公众号)
- **协作**:飞书 / 钉钉 / Notion / 蓝湖
- **市场监测**:新榜 / 蝉妈妈 / 飞瓜 / 第三方平台数据工具

技术深度不需要,但**广度要有**。

### 规则 7:title 重定位

过细分的 title 改成市场可读的:
- "版权运营" → "内容运营"(版权运营太细,招聘官不熟)
- "新媒体助理" → "新媒体运营"
- "活动专员" → "活动运营"
- "用户增长实习生" → "用户增长 / 用户运营"

**职责描述本身不能编**(见 SKILL.md §6 保真原则)。

## 不同企业性质下的运营写法差异

| 企业性质 | 看重什么 | 写法重点 |
|---------|---------|---------|
| **大厂(互联网)** | 量化 + 数据驱动 + 用户洞察 | OKR 思维、AB 测试、数据可视化、增长黑客 |
| **新消费 / 品牌方** | 内容感知 + 用户洞察 + 转化 | 私域运营、爆款拆解、UGC 共创、跨界营销 |
| **MCN / 网红机构** | 流量敏感度 + 内容审美 | 账号孵化、平台算法理解、内容矩阵 |
| **国企新媒体** | 主旋律内容 + 平台合规 | 政务公众号运营、舆情管理、合规审稿 |
| **外企品牌** | brand building + 国际化 | KOL 合作、跨文化内容、品牌资产建设 |

→ 跟所在公司性质的 patterns 文件(`企业性质/<大厂|国企|外企>.md`)叠加使用。

## 常见地雷

1. **写"负责公众号运营"但没有数据** — 没量级就是没说话,起码补"运营期间粉丝增长 / 阅读量增长"
2. **小标题写形容词** — "高效激活"、"精准定制"、"深度洞察",招聘官一秒翻篇
3. **方法论只写名称没说明** — "用 AARRR 模型"等于没说,要加括号写具体怎么用
4. **工具栈堆叠** — 列 10+ 工具显得没专精,3-5 个核心 + 标熟练度即可
5. **不分场景** — 大厂运营和电商运营和私域运营不是一回事,简历要针对具体方向
6. **职责堆叠** — 一段实习写 7-8 条职责,招聘官只看头 2 条。要选 3-4 条最重要 + 量化

## 自我评价写法

**好的运营自我评价**(具体 + 有方法论):
> "3 段互联网运营实习,熟悉用户分层、内容策划、活动复盘全流程,擅长用数据驱动决策。在 song 空间公众号实习期间独立运营内容,3 个月将单篇阅读量从 6000 提升至 3-4 万。"

**烂的运营自我评价**(空话):
> "对运营充满热情,有责任心,擅长沟通协作,有较强的学习能力和抗压能力。"
