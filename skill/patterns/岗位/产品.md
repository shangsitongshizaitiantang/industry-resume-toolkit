# 产品经理 Patterns

涵盖:**B 端产品 / C 端产品 / 数据产品 / SaaS 产品 / 平台型产品**。产品经理是大厂 / 创业公司 / SaaS 公司的核心岗,跟运营、研发、设计紧密协作。

---

## 风格关键词

**用户视角 / 业务理解 / 数据驱动 / 跨部门协作 / 系统思考**

招聘官看产品简历最在意:**你懂不懂用户、能不能定义需求、推不推得动跨部门、有没有数据 sense**。

---

## 核心规则

### 规则 1:用 STAR + 业务价值结构

产品经理的项目经历必须讲完整的故事:

> **背景(S)**:业务遇到 XX 问题 / 用户痛点 / 战略机会
> **目标(T)**:产品需要解决什么(用 KPI 表达)
> **行动(A)**:你做了什么(用户调研 + 需求拆解 + 方案设计 + 推动落地)
> **结果(R)**:量化业务影响(转化提升 X% / DAU 增长 Y / 收入 Z 万)

每段经历用这个完整故事讲透,**不要堆动作**。

### 规则 2:动词要"产品化"

产品经理简历的高质量动词:
- **定义需求**:洞察、抽象、定义、规划、设计
- **推动落地**:推动、协调、对齐、落地、上线
- **数据验证**:验证、AB 测试、复盘、迭代、优化
- **战略思考**:拆解、规划、规模化、商业化

避免:
- ❌ "参与" / "协助" / "学习" — 显得没 ownership
- ❌ "负责" 后面没具体动作

### 规则 3:量化必须"业务级"

产品经理的量化跟运营/开发都不同——要直接挂业务指标:

- **C 端**:DAU / MAU / 留存率 / 转化率 / 用户付费率 / NPS
- **B 端**:客户数 / 续费率 / ARR / 单客户价值 / 客户满意度
- **数据/工具**:覆盖业务方数 / 数据可信度 / 决策影响范围

示例:
> "主导用户成长体系产品改版,上线后**新用户 7 日留存率从 32% 提升至 41%**,**月活提升 18%**,该模式被复制到另外 2 个业务线"

### 规则 4:必须有"用户 / 客户"视角

产品简历最忌讳的是**只讲功能不讲用户**:

- ❌ "上线了 X 功能,使用 Y 技术,实现 Z 效果"
- ✅ "针对 [用户群] 在 [场景] 下 [痛点],设计 X 功能,实现 [用户价值]"

让用户洞察显式出现在简历里。

### 规则 5:跨部门协作要具体

产品经理的核心 soft skill 是"推动跨部门",但**不要写"良好的跨部门沟通能力"**这种空话。

要写具体:
- "协调 5 个研发 + 2 个设计 + 3 个业务方,8 周完成 V1.0 上线"
- "推动算法团队和数据团队联合接入,打通用户画像 → 推荐引擎链路"

### 规则 6:methodology / 方法论加分

产品经理 JD 经常看的方法论关键词:
- **用户调研**:深度访谈 / 用户旅程地图 / 用户画像 / 焦点小组
- **需求管理**:KANO 模型 / RICE 优先级 / MoSCoW
- **设计思维**:Design Thinking / 双钻模型 / Jobs to be Done
- **数据驱动**:AB 测试 / 漏斗分析 / 北极星指标 / OKR
- **B 端思维**:PMF / Product-Market Fit / 客户旅程 / SaaS 指标

**真用过的就写**,没用过的别堆。

### 规则 7:工具栈要"产品人专属"

产品经理简历的工具栈:
- **原型 / 设计**:Figma(必须熟练)、Sketch、墨刀、Axure
- **数据**:SQL(基础查询必备)、Excel、Tableau / Power BI、神策 / GrowingIO
- **协作**:Notion / 飞书 / Jira / Trello / ClickUp
- **AI / 高阶**:Cursor / Claude / ChatGPT(现在很多公司看 AI tooling 能力)

### 规则 8:title 重定位

- 小公司"产品助理" → 简历直接写"产品经理(助理岗)"或"初级产品经理"
- "需求分析师" → 在产品方向找工作时,改成"产品经理 / 需求分析"
- "项目经理"做产品工作的 → 改成"产品+项目"双线 title

## 不同企业性质下的产品写法差异

| 企业性质 | 看重什么 | 写法重点 |
|---------|---------|---------|
| **大厂 C 端** | 量化 + 用户洞察 + 实验思维 | DAU/MAU、AB 测试、北极星指标 |
| **大厂 B 端** | 业务理解 + 系统思考 + 跨团队 | 客户价值、SLA、流程优化 |
| **SaaS 公司** | PMF + 续费率 + 客户旅程 | NRR / GRR、客户成功故事 |
| **传统行业数字化** | 业务流程理解 + 推动落地 | 流程改造、降本增效、组织变革 |
| **外资科技公司** | 英文表达 + 国际化 + Discovery | STAR 法、Discovery、Hypothesis-driven |

## 常见地雷

1. **简历像产品需求文档** — 列功能列模块,没业务价值,招聘官看了像看 PRD 流水账
2. **没有数据** — 产品经理简历没数字 = 没做过事
3. **过度依赖术语** — 堆"用户分层"、"AARRR"、"OKR" 但没有具体案例
4. **跨部门写得太抽象** — "推动跨部门协作"换成具体数字和事
5. **混淆做和负责** — 产品经理是"定义 + 推动",不是亲手写代码 / 做设计,要分清楚
6. **应届产品堆校园项目** — 5 个 dummy 项目不如 1 个深度复盘的真实尝试

## 校招 / 实习产品经理特别建议

应届 / 实习产品经理简历:
- **互联网 + 副业经历**比啥都强(自己做过小程序 / 公众号 / 内容账号)
- **产品分析报告**可以作为项目(竞品分析 + 用户体验地图 + 改进建议)
- **创业项目**含金量很高,哪怕是校园级别
- **AI 工具产品化经历**(用 Coze / Dify / GPTs 做了东西)是新加分项

## 自我评价写法

**好的产品自我评价**:
> "2 段互联网产品实习(腾讯、字节),熟悉 C 端产品全流程,擅长用数据驱动决策。在腾讯实习期间主导用户成长体系改版,新用户 7 日留存提升 9pp。掌握 Figma、SQL、AB 测试,持续关注 SaaS 和 AI 产品赛道。"

**烂的**:
> "热爱产品工作,有责任心,有较强的逻辑思维和用户思维。"
