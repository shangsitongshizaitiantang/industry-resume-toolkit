# 数据分析 Patterns

涵盖:**业务数据分析师 / 商业分析师 / BI 分析师 / 增长分析师 / 行业研究员**。跨行业需求大,互联网 / 金融 / 咨询 / 零售 / 制造都需要。

---

## 风格关键词

**数据严谨 / 业务理解 / 工具熟练 / 洞察输出 / 结构化思考**

招聘官看数据分析简历最在意:**你的数据准不准、业务理解深不深、能不能讲故事**。

---

## 核心规则

### 规则 1:每段经历都要有"数据规模"

数据分析师简历必须显式量化处理的数据规模:
- "**处理 5000 万+ 条用户行为数据**,通过 SQL + Python 完成清洗与建模"
- "**分析 200+ 家门店运营数据**,识别影响 GMV 的 12 个核心因子"
- "**搭建 30+ 张数据报表**,覆盖产品、运营、市场 3 个部门"

数据量级是数据分析师的硬通货。

### 规则 2:工具栈要分层

数据分析师简历的工具栈分三层,**每层都要标熟练度**:

```
数据提取:SQL(精通 — 复杂 JOIN / 窗口函数 / CTE)、Excel(精通 — 透视表 / VLOOKUP / 公式)
数据处理:Python(熟练 — pandas / numpy / scikit-learn)、R(基础)
数据可视化:Tableau / Power BI(精通)、ECharts、Matplotlib / Seaborn
统计建模:A/B 测试、回归、聚类、分类、时间序列
业务工具:神策 / GrowingIO / Google Analytics / 阿里量子统计
```

**不要堆 10+ 工具**,3-5 个核心 + 标熟练度。

### 规则 3:经历要"分析过程 + 业务结论"双结构

数据分析简历每个项目都用这个结构:

> **业务问题**:XX 业务遇到 YY 问题 / 数据异常
> **分析方法**:用 [SQL / Python / 模型] 处理 [数据规模],应用 [方法 — 漏斗 / 多维拆解 / 回归]
> **核心发现**:发现 [量化结论,eg 主要影响因子 X 占比 60%]
> **业务影响**:基于此推动 [业务动作],带来 [量化结果]

示例:
> "**用户流失分析项目**:针对月活下滑 12% 的现象,用 SQL 拉取 3 个月 500w+ 用户行为数据,通过漏斗 + 多维拆解发现 24-30 岁女性用户在 [关键场景] 流失率最高(达 35%)。推动产品改进登录引导流程,流失率从 35% 降至 22%,月活回升 8%。"

### 规则 4:统计 / 模型术语用对

数据分析的术语滥用是大忌。要用就用对:
- **A/B 测试** ≠ 对比分析(A/B 测试要有显著性检验、样本量、置信区间)
- **回归分析** ≠ 趋势分析(回归要写清楚是线性 / 逻辑 / 多元)
- **聚类** ≠ 分类(聚类是无监督,分类是有监督)
- **相关性** ≠ 因果性(简历里不要把相关性说成因果)

简历里写的统计方法,**面试时会被追问**,确认懂了再写。

### 规则 5:业务理解比技术更重要(常被忽视)

新手数据分析师常把简历写成"我会 X 工具",而招聘官真正看的是"你懂业务吗"。

**业务理解的体现**:
- 用业务语言描述项目(不是用数据语言)
- 项目结论挂业务指标(GMV / 留存 / 客单价)而不是技术指标(F1 / AUC)
- 列出你深入了解的业务场景(eg "熟悉电商大促 / SaaS 续费 / 互金风控 业务逻辑")

### 规则 6:报告输出能力要体现

数据分析师不只是查数,还要**输出报告 + 给建议**:

简历里要有"输出"动作:
- "**输出周度 / 月度业务分析报告**,被运营总监采纳作为 X 决策依据"
- "**搭建 BI 看板**,服务于产品、运营、市场 3 个团队,日均使用 50+ 次"
- "**撰写专题分析**,提出 3 条优化建议被 2 条采纳"

报告影响力是数据分析师的核心价值证明。

### 规则 7:实习经历要包装成"分析项目"

数据分析师的实习 / 项目经历都要包装成完整分析项目:

> **项目名称**:[XX 业务] 的 [YY 分析]
> **业务背景**:[1-2 句]
> **数据**:[来源 + 规模]
> **方法**:[工具 + 方法论]
> **结论**:[3-5 条核心发现]
> **影响**:[业务动作 + 量化效果]

零散的"拉数据""做报表"不算项目,要包装成有头有尾的事。

## 不同企业性质下的数据分析写法差异

| 企业性质 | 看重什么 | 写法重点 |
|---------|---------|---------|
| **大厂(互联网)** | SQL / Python / AB 测试 + 业务理解 | 用户行为分析、增长实验、产品迭代支持 |
| **咨询公司** | 结构化思维 + 行业研究 + presentation | 行业建模、案例分析、报告输出 |
| **金融 / 投行** | 严谨 + 财务建模 + 风险量化 | 估值模型、风险评估、合规分析 |
| **快消 / 零售** | 业务 sense + 落地能力 | 选品分析、门店运营、CRM 数据 |
| **外企** | 英文表达 + Statistical methods | A/B testing rigor、CRM segmentation |

## 常见地雷

1. **简历像"工具列表"** — 我会 SQL Python R Tableau Excel 各种工具,但没具体项目 → 招聘官一秒翻篇
2. **数据规模空缺** — "处理了大量数据" → 改成具体多少条 / 多少 GB
3. **统计术语用错** — 把"对比 A 组和 B 组的均值"说成"做了 A/B 测试" → 面试一问就漏
4. **没有业务结论** — "做了分析,输出报告" → 报告说了什么?业务怎么用?
5. **过度堆 ML / AI** — 应聘业务数据分析师却堆深度学习项目 → 跟岗位不匹配
6. **项目过深** — 应届简历不要写"我用 Transformer 做了 X" 这种,大厂会觉得吹

---

## 校招 / 实习数据分析师特别建议

- **Kaggle 比赛**经历加分(写排名 + 用到的方法)
- **课程项目**包装成业务项目(eg "用 XX 大学课程 + 公开数据集做 YY 业务分析")
- **学校数据课题**找跟业务相关的(eg 帮校内某机构做用户分析)
- **自学证书**:Coursera / Datacamp / 阿里云数据分析师证书

## 自我评价写法

**好的**:
> "应用统计学硕士,3 段数据分析实习(美团、招行、Big 4),熟悉用户行为分析、A/B 测试、商业建模全流程。精通 SQL / Python(pandas, sklearn),擅长 Tableau。在美团实习期间独立完成 [项目名],推动业务指标 X 提升 Y%。"

**烂的**:
> "热爱数据,擅长 Excel、SQL、Python,有较强的学习能力和逻辑思维。"
